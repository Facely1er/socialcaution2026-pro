Fix recurring translation toggle issue with improved event handling

This fix addresses a long-standing recurring issue where the language
selector dropdown in the header would not work correctly.

Root causes fixed:
- Race condition in click-outside detection (click vs mousedown)
- Redundant ref management (removed containerRef)
- Low z-index causing dropdown to be hidden (60 -> 9999)
- Missing explicit button types
- Poor UX timing (close dropdown before language change instead of after)

Technical changes:
- Changed event listeners from 'click' to 'mousedown'/'touchstart'
- Removed unnecessary containerRef (kept only buttonRef and dropdownRef)
- Increased z-index from z-[60] to z-[9999]
- Added type="button" to all buttons
- Close dropdown immediately before calling changeLanguage()
- Simplified click-outside logic with early return
- Added preventDefault() to event handlers
